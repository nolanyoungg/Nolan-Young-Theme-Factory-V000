Create a complete, polished website theme for a fictional web design and software development company called Northstar Web Works.

Northstar Web Works builds custom WordPress websites, WooCommerce stores, business automations, analytics dashboards, AI-assisted workflows, and ongoing website maintenance plans for small businesses, service companies, ecommerce stores, and local brands.

The finished website should feel like a real, high-quality agency website. It should not feel like a generic SaaS landing page, a starter template, a wireframe, or a placeholder demo.

Primary audience:
- Small business owners who need a professional website
- Local service companies that need leads
- Ecommerce businesses using WooCommerce
- Companies with outdated WordPress sites
- Business owners interested in automation, analytics, and AI tools
- Teams that need ongoing website support

Primary goals:
- Make visitors trust the company quickly.
- Clearly explain services.
- Show that the company can handle both design and technical implementation.
- Encourage users to request a consultation.
- Present the brand as practical, modern, reliable, and technically capable.
- Make the site feel complete enough to show to a real client.

Brand personality:
- professional
- sharp
- modern
- reliable
- technical but approachable
- confident without sounding arrogant
- clean but not boring
- conversion-focused
- detail-oriented

Visual direction:
Create a premium web agency look with a strong technical edge. Avoid generic startup gradients and overused fake “AI company” visuals.

The design should feel:
- modern
- structured
- polished
- high-contrast where useful
- clean
- slightly futuristic
- trustworthy
- practical
- business-focused

Suggested palette:
- deep navy or near-black
- clean white
- soft gray
- electric blue accent
- bright green or cyan accent used carefully
- subtle grid/panel backgrounds

Use visual treatments that suggest web systems, dashboards, code, interface cards, analytics, and automation without using remote images.

Typography direction:
Use a clean system font stack. Headings should feel bold and confident. Body copy should be highly readable. Code-like labels, badges, and small UI details can use a monospace system stack where appropriate.

Navigation:
Use a polished header with the Northstar Web Works wordmark and these primary nav items:
- Services
- Work
- Process
- Resources
- Contact

Include a strong CTA such as:
- Book a Strategy Call
- Start a Project
- Request a Website Review

The mobile navigation should be clean, usable, and polished.

Homepage content direction:

Hero:
Use a strong headline:
“Websites, systems, and automations built for businesses that need results.”

Use supporting copy that explains Northstar Web Works designs, builds, fixes, and maintains WordPress websites and digital systems for growing businesses.

Include two calls to action:
- Book a Strategy Call
- View Services

The hero should include a visually rich local design treatment inspired by website dashboards, code panels, analytics cards, or interface mockups. Do not use remote images. Do not use gray placeholder boxes.

Trust bar:
Add a compact trust section with credibility markers such as:
- WordPress Development
- WooCommerce Builds
- Analytics & Reporting
- Automation Workflows
- Maintenance Support

Brand statement:
Create a short section explaining that the company combines design, development, analytics, and automation into practical digital systems that help businesses operate better.

Services section:
Include at least six major services:
- Custom Website Design
- WordPress Development
- WooCommerce Stores
- Website Maintenance
- Analytics & Reporting Dashboards
- AI & Automation Integrations

Each service should include:
- clear title
- practical description
- who it helps
- outcome-focused benefit
- link-style CTA

Featured work section:
Create three believable fictional project cards:
- Local Service Website Rebuild
- WooCommerce Plant Shop Optimization
- Operations Dashboard for a Field Team

Each project should include:
- project type
- challenge
- solution
- result-style summary
- visual treatment
- CTA

Do not make exaggerated claims like “3000% growth.” Keep results believable.

Process section:
Include a clear process:
- Discover
- Plan
- Design
- Build
- Launch
- Improve

The process should feel professional and repeatable.

Technical strengths section:
Create a section that highlights technical capabilities:
- clean WordPress architecture
- responsive front-end development
- performance-minded builds
- conversion tracking
- form and CRM workflows
- reporting dashboards
- secure maintenance practices

Make this section feel technical but understandable to business owners.

Maintenance section:
Include a dedicated section for ongoing website maintenance:
- updates
- backups
- monitoring
- troubleshooting
- content edits
- performance checks
- plugin/theme support

This should feel like a real service offer.

AI and automation section:
Include a practical AI/automation section. Focus on real business use cases:
- internal workflow helpers
- customer inquiry routing
- content drafting workflows
- reporting summaries
- form-to-email automations
- lead handling improvements

Avoid overhyping AI. Make it practical.

Testimonials section:
Include three believable fictional testimonials from business owners or managers. They should sound specific and grounded.

Resources/blog preview:
Include three article previews:
- What to Fix Before Redesigning Your Website
- How to Track Better Website Leads
- When a WordPress Site Needs Maintenance

Final CTA:
End the homepage with a strong conversion section inviting visitors to schedule a strategy call or request a website review.

Footer:
Create a complete footer with:
- company name
- short brand statement
- services links
- contact information
- newsletter signup
- social links such as LinkedIn, GitHub, and YouTube
- copyright

Page content direction:

Services page:
Create a complete services overview page with clear service sections, comparison-style service cards, process notes, and a CTA.

Single service page:
Create a reusable single-service layout that can explain one service in depth. Include:
- service hero
- who it is for
- common problems solved
- deliverables
- process
- outcomes
- FAQ-style content
- CTA

Work page:
Create a polished work/portfolio page with project cards, categories, case-study style summaries, and local visual treatments.

Blog/resources page:
Create an editorial resources page with article cards about websites, WordPress, analytics, automation, ecommerce, and maintenance.

Contact page:
Create a strong project inquiry page with:
- intro copy
- inquiry form presentation
- contact details
- service interest options
- expected response time
- FAQ-style notes

About page:
Create a focused about/company page explaining the company’s approach, values, technical standards, and client working style.

Policy page:
Create a simple but styled policy page that looks intentional and complete.

Visual asset direction:
Use local SVGs, CSS gradients, code-inspired panels, dashboard cards, analytics graphics, abstract grid systems, browser-window mockups, and UI-style illustrations.

Visuals should feel custom and relevant to a web development company.

Do not use gray placeholder rectangles.

Do not use remote image URLs.

Do not use CDN assets.

Content rules:
Do not use lorem ipsum.
Do not use “placeholder,” “sample,” “example,” “TODO,” or “coming soon.”
Do not use fake exaggerated claims.
Do not use generic agency filler.
Make the writing clear, direct, credible, and specific.
Write like a real web development company that understands small business needs.

Interaction and polish expectations:
The site should feel responsive, smooth, and carefully finished.
Buttons should have hover and focus states.
Cards should feel clickable where appropriate.
Forms should be styled.
The header should work well on mobile.
The footer should feel complete.
Utility pages should not feel forgotten.
The design should work well on desktop, tablet, and mobile.

Overall quality bar:
The finished result should feel like a real, impressive web development company website that could be shown to a prospective client.

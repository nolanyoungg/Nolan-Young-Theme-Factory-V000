# Codex Fixer Pass

Use this only after validation has failed and the user explicitly approves another Codex run.

- Read the validation output first.
- Make the smallest complete fix set that clears the failures.
- Keep the existing theme direction intact.
- If validation reports missing preview files, Nolan-menu attributes, local images, or template parts, add the required complete implementation rather than suppressing the check.
- Do not weaken validation to pass a bad theme.
- Rebuild, repackage, and revalidate after any compiled or packaged change.

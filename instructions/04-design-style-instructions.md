# Design Style Instructions

## Technical Rules
- Design for the prompt's domain and audience.
- Use responsive layouts, durable spacing scales, and accessible focus states.
- Keep all fonts, icons, images, CSS, and JavaScript local.

## Required File Behavior
- SCSS partials must have meaningful responsibility matching their filenames.
- Compiled CSS must include layout, typography, components, pages, forms, navigation, and responsive rules.
- JavaScript must support navigation and interactions without breaking no-JS content.

## Output Requirements
- Provide source SCSS, compiled CSS, source JS, compiled JS, and local asset notes.
- Make the WordPress theme and static preview feel visually connected.

## Failure Conditions
- Generic unpolished visual system.
- Unreadable contrast, missing focus states, or broken mobile layout.
- Remote font, icon, script, or image dependencies.

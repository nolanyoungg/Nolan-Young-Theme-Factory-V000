# Static Preview Builder

Create the GitHub Pages preview.

- Mirror the WordPress theme visually using only local HTML, CSS, JS, and committed assets.
- Do not rely on PHP, WordPress, remote images, CDNs, or external fonts.
- Keep the preview close enough that it can be used for visual review.
- Create all seven required preview pages and make the header links click between them.
- Match the WordPress templates closely: same structure, classes, order, copy style, imagery, and footer.
- Implement Nolan-menu desktop and mobile behavior in preview JS.
- Update the gallery index so the new theme is discoverable.
